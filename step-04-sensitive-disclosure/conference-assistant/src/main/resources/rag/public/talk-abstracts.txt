Title: Building Secure AI Agents with Quarkus
Speaker: bob

AI agents are powerful but introduce new attack surfaces. This talk walks through building a conference assistant agent using Quarkus and LangChain4j, then systematically hardening it against prompt injection, unauthorized tool use, and data leakage. You will leave with concrete patterns for guardrails, token propagation, and scoped RAG retrieval that you can apply in your own projects.

Title: Reactive Microservices in Practice
Speaker: carol

Reactive programming has moved from a buzzword to a production necessity for services that need to handle thousands of concurrent connections. This session covers practical reactive patterns with Quarkus and Mutiny: backpressure, circuit breakers, retry, and how to avoid common pitfalls that turn reactive code into a debugging nightmare. Live coding throughout.

Title: LangChain4j Deep Dive
Speaker: alice

LangChain4j is the leading Java library for building LLM-powered applications. This deep dive covers the full feature set: chat models, embedding models, RAG pipelines, tool calling, MCP integration, memory management, and testing strategies. Lots of code, few slides.

Title: JVM Performance Tuning on JDK 25
Speaker: dave

JDK 25 ships with Project Loom, Project Panama, and continued improvements to the GC subsystem. This talk examines what actually matters for throughput and latency in production services, how to measure it correctly, and which JDK 25 features deliver the biggest gains. Benchmarks, flame graphs, and GC logs analyzed live.

Title: Zero-Trust Architecture for Microservices
Speaker: alice

Zero trust means every request is authenticated and authorized regardless of network origin. This session covers how to implement zero trust in a Quarkus microservices system: OIDC token propagation between services, service-to-service mTLS, policy-based authorization with OPA, and observability to detect violations. Title: Cloud-Native Observability Without the Overhead
Speaker: carol

Observability is not optional in production, but it has a cost. This talk shows how to get actionable traces, metrics, and logs out of Quarkus services without adding significant latency or resource overhead. Covers OpenTelemetry, Micrometer, structured logging, and how to correlate signals across service boundaries.
